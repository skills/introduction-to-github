This directory contains deployment records for ScrollVerse smart contracts.

| Contract | Description | File |

## Contracts

### ProtocolRegistry (Security Layer 2.0)
Governance registry for ScrollVerse ecosystem protocols with lifecycle management and DAO-enforced transitions.

## File Format

Deployment files are named: `{contract}-{network}-{chainId}.json` or `{network}-{chainId}.json`

Examples:
- `protocol-registry-sepolia-11155111.json`
- `sepolia-11155111.json`

## Deployment Order

Contracts must be deployed in this specific order:

1. **MirrorToken** - No dependencies
2. **MirrorTokenVesting** - Requires MirrorToken address
3. **ScrollVerseTimelock** - No dependencies
4. **ScrollVerseDAO** - Requires MirrorToken and Timelock addresses
5. **MirrorStaking** - Requires MirrorToken address
6. **PharaohConsciousnessFusion** - No dependencies
7. **AnchorManifest** - No dependencies

## Post-Deployment Configuration

After deployment, execute these configuration steps:

1. Grant PROPOSER_ROLE to DAO on Timelock
2. Execute initial token distribution
3. Add beneficiaries to vesting contract
4. Add rewards to staking contract
5. Set staking/DAO addresses on NFT contract
